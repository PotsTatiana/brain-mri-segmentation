# Сегментация МРТ головного мозга (Brain MRI Segmentation)

Нейросетевая система для автоматической сегментации патологических участков (опухолей, очагов) на МРТ-изображениях головного мозга. Включает десктопное приложение с графическим интерфейсом для врачей-рентгенологов.

## О проекте

Система решает задачу семантической сегментации патологий на многоканальных МРТ-срезах в формате HDF5 (2 канала, 256×256). В отличие от базовых решений, использующих стандартные архитектуры без постобработки, данная система комбинирует:

* современную архитектуру **U‑Net++** с предобученным энкодером **EfficientNet‑B4** и трансферным обучением;
* морфологическую постобработку (удаление шумовых компонентов);
* интерактивный инструмент ручной коррекции маски кистью;
* трёхмерную визуализацию патологического участка;
* формирование клинического PDF‑отчёта.

Достигнутое значение Dice на валидационной выборке – **0,9012**, что превышает клинически значимый порог (>0,85).

## Ключевые возможности системы

* **Поддержка нескольких пациентов** – загрузка произвольного количества HDF5-файлов, переключение между ними с сохранением текущего среза.
* **Автоматическая сегментация** – инференс обученной модели на выбранном срезе с регулируемым порогом чувствительности (слайдер от 0 до 1).
* **Ручная коррекция маски** – квадратная кисть переменного размера (1–20 пикселей) для рисования/стирания, поддержка Undo/Redo.
* **Панорамирование и масштабирование** – правая кнопка мыши – панорамирование, колёсико – зум, левая кнопка – рисование (в активном режиме).
* **3D-визуализация** – построение изоповерхности по стеку масок с учётом размера вокселя (библиотека vedo).
* **Экспорт масок для дообучения** – сохранение пар «МРТ + исправленная маска» в формате `.npy` для дальнейшего улучшения модели.
* **PDF‑отчёт** – одностраничный документ (A4) с изображениями среза, техническими параметрами, площадью патологии в пикселях и Dice‑коэффициентом.

## Архитектура проекта

* `gui.py` – главное окно и логика взаимодействия (PyQt5)
* `graphics_view.py` – кастомный виджет с поддержкой зума, панорамирования и рисования
* `hdf5_loader.py` – загрузка многоканальных МРТ-файлов в формате HDF5
* `visualization_3d.py` – построение трёхмерной модели патологического участка (vedo)
* `predictor.py` и `model_loader.py` – загрузка весов `best_model.pth` в архитектуру U‑Net++ и выполнение инференса
* `inference_thread.py` – (опционально) фоновая сегментация всех срезов

## Функциональность

* **Предобработка данных** – нормализация, фильтрация пустых срезов, разделение на train/val/test (70/15/15).
* **Обучение модели** – комбинированная функция потерь (BCE + Dice), оптимизатор AdamW, полиномиальное снижение скорости обучения, ранняя остановка.
* **Постобработка маски** – морфологическое закрытие/открытие, удаление компонентов менее 30 пикселей.
* **Визуализация** – четыре окна: исходное МРТ (с выбором канала DWI/ADC), истинная маска, предсказание, оверлей.
* **Интерактивная коррекция** – кисть с квадратной формой, история изменений, отмена/повтор действий.

## Структура данных

Проект использует набор МРТ-данных в формате HDF5, где каждый файл содержит:
* `images` – массив размерностью (N, 2, 256, 256) – N срезов, 2 канала;
* `masks` – массив размерностью (N, 1, 256, 256) – эталонные маски (опционально).

*Внимание:* исходные HDF5-файлы не включены в репозиторий из-за большого размера. Вы можете использовать собственные данные, приведённые к формату (N,2,256,256).

1. Клонируйте репозиторий:
   ```bash

   cd brain-mri-segmentation

2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   source venv/bin/activate   # Linux/macOS
   venv\Scripts\activate      # Windows

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt

4. Поместите файлы с весами модели best_model.pth и шрифт arial.ttf в корневую папку проекта.

5. Запустите приложение:
   python main.py
